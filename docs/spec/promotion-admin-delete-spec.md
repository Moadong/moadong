# 홍보 게시글 삭제 기능 스펙

## 목적

개발자 포털에서 선택한 홍보 게시글을 삭제할 수 있어야 한다.

현재 삭제는 사용자 경험 기준으로는 "삭제"지만,
백엔드 저장 방식은 소프트 딜리트와 연결된다.

## 대상 범위

- 개발자 포털 홍보 게시판 관리 화면
- 홍보 게시글 삭제 API
- 삭제 후 목록/선택 상태 정리

## 사용자 시나리오

1. 관리자가 개발자 포털에서 홍보 게시글 목록을 조회한다.
2. 목록에서 게시글 하나를 선택한다.
3. 편집 패널에서 `삭제` 버튼을 누른다.
4. 확인 창에서 삭제를 승인한다.
5. 게시글은 목록에서 사라지고 편집 상태는 해제된다.

## UI 스펙

대상 파일:

- `backend/src/main/resources/static/dev/index.html`

### 액션 버튼

편집 패널의 `promotion-actions` 영역에 아래 버튼 순서를 유지한다.

1. `저장`
2. `원본으로 되돌리기`
3. `삭제`
4. `선택 해제`

### 삭제 버튼 활성화 조건

삭제 버튼은 아래 조건을 모두 만족할 때만 활성화된다.

1. create 모드가 아니다.
2. 게시글이 선택되어 있다.
3. 로딩/저장/업로드/삭제 진행 중이 아니다.

### 삭제 중 상태

삭제 진행 중에는 아래 동작이 잠긴다.

1. 입력 폼 수정
2. 저장
3. 새 글 작성
4. 목록 새로고침
5. 이미지 업로드
6. 선택 해제

버튼 텍스트는 `삭제 중...`으로 바뀐다.

### 확인 창 문구

기본 문구:

- `"{제목}"을(를) 정말 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다.`

저장되지 않은 변경이 있을 때:

- `저장되지 않은 변경사항이 있습니다. "{제목}"을(를) 정말 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다.`

## API 스펙

### 엔드포인트

- `DELETE /api/promotion/{articleId}`

### 권한

- `DEVELOPER`

### 성공 응답

- HTTP `200`
- 메시지: `홍보 게시글이 삭제되었습니다.`

### 실패 응답

- HTTP `403`
  - 메시지: `개발자 계정으로 로그인하세요.`
- HTTP `404`
  - 에러 코드: `902-1`
  - 메시지: `홍보 게시글이 존재하지 않습니다.`
- 그 외 실패
  - API message 우선 사용

## 프론트 처리 스펙

### 성공 시

1. 목록을 다시 조회한다.
2. 선택 상태를 해제한다.
3. 편집 폼을 초기화한다.
4. 성공 메시지를 표시한다.
5. 토스트 `홍보 게시글 삭제 완료`를 표시한다.

### 404 시

1. 해당 게시글을 로컬 목록에서 제거한다.
2. 선택 상태를 해제한다.
3. `선택한 홍보 게시글을 찾을 수 없습니다. 목록을 새로고침하세요.` 또는 서버 메시지를 보여준다.

### 목록 재동기화 실패 시

삭제 자체가 성공했다면 성공 메시지는 유지한다.

추가 경고:

- `삭제는 완료되었지만 목록 동기화에 실패했습니다. 목록을 새로고침해 확인하세요.`

## 상태 관리 스펙

프론트 상태 변수:

- `promotionIsDeleting`

이 값은 삭제 요청 시작 시 `true`,
완료 또는 실패 후 `false`로 돌아가야 한다.

## 검증 기준

아래가 모두 만족되면 기능이 정상이다.

1. 수정 모드에서만 삭제 버튼이 활성화된다.
2. create 모드에서는 삭제 버튼이 비활성화된다.
3. 삭제 후 목록에서 게시글이 사라진다.
4. 삭제 후 선택 상태가 해제된다.
5. 403, 404, 일반 실패가 의도한 메시지로 처리된다.
